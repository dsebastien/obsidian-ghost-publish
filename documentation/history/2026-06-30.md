# 2026-06-30

## Feature — Panel search / filter box

Added a search field at the top of the side panel that filters the active sub-tab's list (Triage / Queue / Recently published). Behaves like the tools-website search (typo-tolerant fuzzy matching).

### What was done

- **`src/app/utils/fuzzy-search.fn.ts`** — ported the tools-website fuzzy search (`@leeoniya/ufuzzy` + subsequence fallback, per-field weights, start-of-field / substring / exact-match bonuses). Pure, no DOM/Obsidian dep. New dependency: `@leeoniya/ufuzzy@1.0.19` (~7 KB, browser-safe, bundled into `main.js`). `fuzzy-search.spec.ts` — 9 cases.
- **`src/app/views/pages/note-search.ts`** — `filterNotesBySearch(items, query, getTitle, getPath)` wrapper: title (weight 5) + path (weight 2); empty query is a no-op (returns the original list/order). `note-search.spec.ts` — 5 cases.
- **`view-state.ts`** — added `searchQuery` to `ViewState` (panel-local, persists across sub-tab switches, reset on re-open).
- **`ghost-publish-view.ts`** — search box rendered in the header (icon + `<input type="search">` + clear button). On input it updates `state.searchQuery` and debounce-re-renders **only** `.gp-view-content` via new `rerenderContent()` (debounce `SEARCH_RENDER_DEBOUNCE_MS = 120`), so the input keeps focus + caret. Extracted `renderActivePage()` and `resolveActivePreset()` to share between full render and content-only render. Timer cleared on `onClose`.
- **triage / queue / recently-published pages** — each takes a `searchQuery` param and filters via `filterNotesBySearch`; summaries and empty states are search-aware ("No … match your search."). Recently-published filters the **full** set before the 30-item cap so older matches stay findable. Queue search filters only the visible list — Sync still acts on the full queue (BR-UI-7).
- **`styles.src.css`** — `.gp-search-*` styles (form-field-styled wrap, focus-within accent border, native search cancel button hidden, `.gp-hidden` utility).

### Key decisions

- **Reused the tools-website implementation verbatim** (the user pointed at it) rather than hand-rolling — same ranking behaviour, small dep. uFuzzy is tiny and browser-safe, so it clears the "avoid large dependencies" bar.
- **Content-only re-render** instead of full `render()` on keystroke: a full re-render recreates the header and would blur the input every character. Search box lives in the header (untouched by `rerenderContent`).
- **Query persists across sub-tabs**: one search box that filters "the active view" reads most naturally if the term survives a tab switch.
- Documented as **BR-UI-8**.
