# 2026-05-19

## Refactor — generic Ghost publishing plugin with presets

Renamed `ghost-publish-news` (id) / `Ghost Publish News` (name) / `obsidian-ghost-publish-news` (package) → `ghost-publish` / `Ghost Publish` / `obsidian-ghost-publish`. Removed every author-specific default (Ghost URL, newsletter slug, news feed note path, hardcoded wikilink map) and made the entire pipeline preset-driven.

## Why

The first iteration was hardwired around a single news-publishing workflow. The new shape exposes:

- **Presets** as a top-level concept — one panel tab per preset (Blog post / News / Microblog / …).
- **Configurable frontmatter property names** (one global set, since a note can only belong to one preset at a time).
- **Auto-fetched tag and newsletter lists** from Ghost, cached + manually refreshable.

## Key decisions

- **Single global FM keys** (BR-PRESET-1). A note has at most one preset assignment. Reason: the user wants "either/or" — much simpler model and one Ghost post per note.
- **Per-preset tabs + sub-tabs** in the panel. Top row = enabled presets; below = Triage / Queue / Recently published for the active preset.
- **Empty state with "Open settings"** when no Ghost config OR no presets enabled. Reuses one component for both surfaces.
- **Manual refresh only** for tags/newsletters cache. Settings tab shows last-fetched timestamps + a button. No auto-fetch.
- **Tag input uses `AbstractInputSuggest`** for autocomplete, ordered list with up/down/remove. Newsletter is a dropdown of cached newsletters; the saved slug is preserved even if it's not in the cache (shown as "(uncached)").
- **Author-agnostic defaults** (BR-DEF-1). `presets: []`, empty stripSections, empty knownUrls, `eligibility: ''` etc.
- **Canonical URL becomes per-preset** (`canonicalUrlEnabled`). Some presets want it (News with public mirror); others don't (drafts, microblogs).
- **Listing note becomes per-preset**. Path is per-preset; format is the same minimalist markdown listing.

## Open questions

- Preset reordering uses up/down buttons. A drag handle would be nicer but adds complexity (HTML5 DnD + scrolling). Defer until needed.
- The known-URL map is global. Some users may want per-preset overrides — defer until requested.
- The plugin currently calls `crypto.randomUUID()` for preset ids; falls back to `Date.now() + Math.random()` if missing. Should be safe in modern Obsidian (Electron) but verify on minimum supported version.

## Later in the day — 0.1.1 + post-release fixes

After the initial scaffold landed, several user-reported issues + polish iterations went in:

- **Build script** read `PLUGIN_ID` from `package.json` (which keeps the `obsidian-` prefix) instead of `manifest.json` (which doesn't, per catalog rules). Plugin landed in the wrong vault folder until fixed.
- **Scroll preservation** (commit `cd6318e`, `fix(plugin): preserve scroll on triage and queue actions`): two-layer defence — in-place card removal via `pages/card-animations.ts` for single-card actions (publish / publish+email / ignore / remove-from-queue), `snapshotScroll` / `restoreScroll` around full re-renders for the refresh button + sync.
- **Listing-note self-ignore** (commit `4b3e081`, `fix(plugin): mark regenerated listing note as ignored`): `regenerateListingNote` now takes `FrontmatterPropertyNames` and calls `processFrontMatter` after the body write to set the configured ignore flag. Documented as `BR-PRESET-4`.
- **Hand cursor + post-sync refresh** (commit `09666ea`): broad `:not([disabled])` cursor rule covering panel, settings, modals; second view refresh scheduled `POST_SYNC_REFRESH_DELAY_MS` (500 ms) after a sync completes so badges + timestamps reflect post-sync state. Documented as `BR-UI-4`, `BR-UI-5`, `BR-UI-6`.

Tagged releases so far: `0.1.0`, `0.1.1`. Unreleased work above goes into the next bump.

## Even later — Queue filter + click semantics

- **Hide synced notes by default in the Queue tab** (commit `feat(plugin): hide synced notes by default in the Queue tab`): added `queueHideSynced: boolean` (default `true`) on `ViewState`; an inline `Show synced notes` checkbox flips it. The Queue page now filters the visible list to `!hasGhostId` when the toggle is off. **Sync button is unaffected** — it always operates on the full queue (re-syncing a synced note is a cheap content-hash no-op). Documented as `BR-UI-7`.
- **Standard click semantics on panel note titles** (commit `fix(plugin): standard click semantics on panel note titles`): titles in Triage / Queue / Recently published used to open in a new tab on every click (`openLinkText(..., true)`). Replaced with a shared `pages/open-note-link.ts` helper using `Keymap.isModEvent` (Ctrl/Cmd + click) + `ev.button === 1` (middle-click). Plain click now opens in the current tab.
- **Spec coverage**: `open-note-link.spec.ts` — five tests covering plain click, Ctrl+click, middle-click, right-click, and other non-actionable buttons. `Keymap` mock added to `test-setup.ts` (tests can override `Keymap.isModEvent` directly; `afterEach` restores the default).
