# 2026-06-02

## Panel — uniform card sizing, path moved to tooltip

Cards across the three panel sub-tabs (Triage / Queue / Recently published) now share a uniform baseline height and bottom-aligned action row. The note path is no longer rendered in the card meta row — it is shown as the `title` attribute on the card title link and surfaces on hover.

## Why

- Card height previously varied per tab and per row because: (1) only Queue cards have a badges row, (2) the meta row used `flex-wrap` so long paths spilled to a second line, (3) actions sat directly after the last content row, with no bottom anchor.
- The full path duplicated info already present on hover-of-the-link and dominated narrow panels.

## Changes

- `src/styles.src.css`
    - `.gp-card`: added `min-h-28` (≈7rem) so every card meets the natural max height (Queue with badges).
    - `.gp-card-actions`: added `mt-auto` so the action row anchors to the bottom regardless of intermediate content.
    - `.gp-card-meta`: removed `flex-wrap`; single-line meta keeps cards uniform.
    - `.gp-card-path`: rule deleted (no longer used).
- `src/app/views/pages/triage-page.ts`, `queue-page.ts`, `recently-published-page.ts`
    - Dropped the `.gp-card-path` span from each card.
    - Added `attr: { title: <file path> }` on the `.gp-card-title` `<a>` so the path is available on hover.
